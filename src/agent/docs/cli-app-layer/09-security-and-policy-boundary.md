# 09. 安全边界与策略职责

## 1. 责任划分

- 内核策略：工具执行前校验（已存在）。
- 应用层策略：执行身份、会话归属、路径白名单注入、日志脱敏。
- CLI 层：参数基本校验与用户确认交互。

## 2. 应用层必须做的防护

- 校验 `conversationId/executionId` 格式。
- 禁止非法路径进入本地存储根目录之外。
- 对日志中的敏感字段做掩码（token/key/password）。
- 避免输出工具原始高危参数到终端。

## 3. 当前延期项说明

- `bash` 安全默认收敛与重写属于独立 P0 工作流。
- 本文档仅定义边界，不替代该重构任务。
