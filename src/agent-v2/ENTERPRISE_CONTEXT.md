## 二、ClickHouse 表结构设计

```sql
-- 消息表 (按会话分区)
CREATE TABLE messages (
    message_id     String,
    session_id     String,
    conversation_id String,
    parent_id      String DEFAULT '',
    role           Enum8('system' = 1, 'user' = 2, 'assistant' = 3, 'tool' = 4),
    content_type   String DEFAULT 'text',
    content        String,
    tool_call_id   String DEFAULT '',
    tool_name      String DEFAULT '',
    metadata       JSON,
    timestamp      DateTime64(3),
    version        UInt32 DEFAULT 1,

    -- 向量 (可选，用于语义搜索)
    embedding      Array(Float32) DEFAULT [],

    -- 聚合列
    tokens         UInt32 DEFAULT 0,

    -- 分区与索引
    INDEX idx_session (session_id) TYPE bloom_filter GRANULARITY 1,
    INDEX idx_timestamp (timestamp) TYPE minmax GRANULARITY 1,
)
ENGINE = MergeTree()
PARTITION BY toYYYYMM(timestamp)
ORDER BY (session_id, timestamp)
TTL timestamp + INTERVAL 30 DAY
SETTINGS index_granularity = 8192;
```

## 六、技术栈汇总

| 层级 | 组件 | 用途 |
|------|------|------|
| **L1 热数据** | Redis Cluster | 实时 Context |
| **L2 温数据** | ClickHouse | 最近 30 天会话 |
| **L3 冷数据** | S3/MinIO | 归档存储 |
| **向量存储** | Milvus/pgvector | Memory 语义搜索 |
| **全文搜索** | Elasticsearch | 关键词检索 |
| **消息队列** | Kafka | 异步写入缓冲 |
