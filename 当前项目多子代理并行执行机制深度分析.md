## 一、执行摘要

本项目通过 **Task V2** 系统实现了企业级的多子代理并行执行机制。系统采用 **DAG（有向无环图）任务调度** + **并发控制** 的架构，支持依赖关系管理、并发限制、故障隔离和事件追踪。

### 核心特性

| 特性 | 实现状态 | 说明 |
|------|---------|------|
| **DAG 任务调度** | ✅ 完整支持 | 支持任务依赖关系和拓扑排序 |
| **并发控制** | ✅ 完整支持 | 可配置的 max_parallel 参数（1-50） |
| **依赖检测** | ✅ 完整支持 | 自动检测循环依赖 |
| **状态持久化** | ✅ 完整支持 | SQLite + JSONL 双层存储 |
| **事件追踪** | ✅ 完整支持 | Append-only 事件日志 |
| **故障隔离** | ✅ 完整支持 | 子代理独立上下文和错误边界 |
| **取消机制** | ✅ 完整支持 | AbortSignal 传播和清理 |

## 三、并行执行核心机制

### 3.1 task_dispatch_ready 工具详解

这是实现并行执行的核心工具，其工作流程如下：

```
┌─────────────────────────────────────────────────────────────────┐
│                 task_dispatch_ready 执行流程                     │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
        ┌────────────────────────────────────┐
        │ 1. 扫描任务 (scan_limit 限制)      │
        │    - 列出所有任务                  │
        │    - 加载依赖关系图                │
        └───────────────┬────────────────────┘
                        │
                        ▼
        ┌────────────────────────────────────┐
        │ 2. 依赖满足性检查                  │
        │    for each task:                  │
        │      if (task.status in ['pending',│
        │          'blocked'] &&              │
        │          all deps completed):      │
        │        promote to 'ready'          │
        └───────────────┬────────────────────┘
                        │
                        ▼
        ┌────────────────────────────────────┐
        │ 3. 过滤候选任务                    │
        │    - status = 'ready'              │
        │    - 无活跃运行                    │
        │    - 依赖全部满足                  │
        └───────────────┬────────────────────┘
                        │
                        ▼
        ┌────────────────────────────────────┐
        │ 4. 并发调度 (max_parallel 限制)    │
        │    selected = candidates.slice(    │
        │        0, max_parallel)            │
        │                                    │
        │    for (task of selected):         │
        │      startRun(task)                │
        └───────────────┬────────────────────┘
                        │
                        ▼
        ┌────────────────────────────────────┐
        │ 5. 等待结果 (如果 wait=true)       │
        │    Promise.all(                    │
        │      dispatched.map(r =>           │
        │        waitRun(r.run_id)))         │
        └────────────────────────────────────┘
```

**关键代码片段**（`src/tool/task-v2-tools.ts` 第 589-667 行）:

```typescript

const isDependencySatisfied = (taskId: TaskId): boolean => {
  const deps = depsByTaskId.get(taskId) ?? [];
  return deps.every((depTaskId) => {
    const depTask = taskById.get(depTaskId);
    return depTask?.status === 'completed';

// 2. 晋升满足依赖的任务
for (const task of tasks) {
  if (task.status !== 'pending' && task.status !== 'blocked') {
    continue;
  }
  if (!isDependencySatisfied(task.id)) {

    status: 'ready',
    expectedVersion: task.version,
  });
  promotedTaskIds.push(promoted.id);
}

// 3. 并发调度（关键！）
const selected = dispatchQueue.slice(0, args.max_parallel);
for (const task of selected) {
  const run = await this.runtime.service.startRun(
    sessionId,
    createRunId(),
    runAdapter.input,
    runAdapter.adapter

  dispatched.push({ task_id: task.id, run_id: run.id });

// 4. 并发等待（如果需要）

  const waits = await Promise.all(
    dispatched.map((item) =>
      this.runtime.service.waitRun(sessionId, ensureRunId(item.run_id), {
```

### 3.2 并发控制策略

#### 3.2.1 max_parallel 参数

- **默认值**: 3
- **范围**: 1-50
- **作用**: 限制同时运行的子代理数量

#### 3.2.2 调度策略

```typescript
// FIFO + 依赖优先
const dispatchQueue: Task[] = [];
for (const task of candidates) {
  const runs = await this.runtime.service.listRuns(sessionId, { taskId: task.id, limit: 20 });
  const hasActiveRun = runs.some((run) => ACTIVE_RUN_STATUSES.includes(run.status));
  if (hasActiveRun) {
    skipped.push({ task_id: task.id, reason: 'active_run_exists' });
    continue;
  }
  dispatchQueue.push(task);
}

// 取前 max_parallel 个
const selected = dispatchQueue.slice(0, args.max_parallel);
```

**调度规则**:
1. 优先调度依赖已满足的任务
2. 跳过已有活跃运行的任务
3. 按 FIFO 顺序选择前 N 个

### 3.3 Agent 执行适配器

**位置**: `src/tool/task-v2/runtime/agent-adapter.ts`

负责创建和执行子代理：

```typescript
export function createAgentRunExecutionAdapter(options: AgentRunAdapterOptions) {
  return {
    execute: async (
      run: Run,
      signal: AbortSignal,
      appendEvent: (event: Omit<RunEvent, 'seq'>) => Promise<void>
    ): Promise<RunExecutionResult> => {
      // 1. 解析输入
      const parsed = parseRunInputSnapshot(run);
      
      // 2. 构建子 Agent
      const agent = new Agent({
        provider: options.provider,
        toolManager: options.createToolManager?.(run) ?? new ToolManager(),
        systemPrompt: buildSystemPrompt(config),
        maxSteps: config.maxSteps,
        plugins: [buildPlugin(run, appendEvent)]
      });
      
      // 3. 执行（支持超时和取消）
      const result = await executeWithTimeoutAndCancel(
        agent.run(parsed.prompt),
        { timeoutMs, signal }
      );
      
      return result;
    }
  };
}
```

**关键特性**:
- **独立的工具管理器**: 每个子代理有自己的工具沙箱
- **事件冒泡**: 子代理事件通过 Plugin 冒泡到父代理
- **超时控制**: 使用 `setTimeout` + `AbortSignal`
- **错误隔离**: 子代理错误不影响其他并行任务

## 六、错误处理与容错机制

### 6.1 错误隔离

**原则**: 每个子代理有独立的错误边界

```typescript
// sqlite-runner.ts
async executeRun(run: Run): Promise<void> {
  try {
    const adapter = this.adapters.get(run.id);
    if (!adapter) {
      await this.finishRun(run, {
        status: 'failed',
        error: 'missing execution adapter',
      });
      return;
    }
    
    const result = await adapter.execute(run, controller.signal, appendEvent);
    await this.finishRun(run, result);
  } catch (error) {
    // 单个任务失败不影响其他并行任务
    await this.finishRun(run, {
      status: 'failed',
      error: toErrorMessage(error),
    });
  } finally {
    this.activeControllers.delete(run.id);
  }
}
```

### 6.2 超时控制

```typescript
// agent-adapter.ts
function executeWithTimeoutAndCancel(
  promise: Promise<AgentResult>,
  options: { timeoutMs?: number; signal: AbortSignal }
): Promise<RunExecutionResult> {
  return new Promise((resolve, reject) => {
    const timer = createTimeoutTimer(options.timeoutMs, onTimeout);
    const handler = createAbortRelay(options.signal, controller);
    
    promise
      .then((result) => resolve(createSucceededResult(result.text)))
      .catch((error) => {
        if (controller.signal.aborted) {
          resolve(createCancelledResult());
        } else {
          resolve(createFailedResult(error));
        }
      })
      .finally(() => {
        clearTimer(timer);
        detachAbortListener(options.signal, handler);
      });
  });
}
```

### 6.3 恢复机制

**场景**: 进程崩溃后重启

```typescript
// sqlite-runner.ts
async recover(): Promise<void> {
  // 扫描所有未完成的运行
  const runs = await this.repo.listRunsByStatus(
    ['queued', 'running', 'cancel_requested'], 
    2000
  );
  
  for (const run of runs) {
    if (run.status === 'running') {
      // 标记为失败（无法恢复运行中的状态）
      await this.finishRun(run, {
        status: 'failed',
        error: 'run interrupted by runner restart',
      });
    } else if (run.status === 'cancel_requested') {
      // 标记为已取消
      await this.finishRun(run, {
        status: 'cancelled',
        error: 'cancelled during recovery',
      });
    } else if (run.status === 'queued') {
      // 如果有适配器，重新启动
      if (this.adapters.has(run.id)) {
        void this.executeRun(run).catch(() => undefined);
      } else {
        await this.finishRun(run, {
          status: 'failed',
          error: 'queued run converged without execution adapter',
        });
      }
    }
  }
}
```

## 八、最佳实践

### 8.1 任务设计原则

1. **单一职责**: 每个任务应该有明确单一的目标
2. **合理粒度**: 任务不应过大（超时）或过小（调度开销）
3. **明确依赖**: 显式声明任务间依赖关系

### 8.2 并发控制建议

| 场景 | 推荐 max_parallel | 理由 |
|------|------------------|------|
| **CPU 密集型** | 2-4 | 避免资源竞争 |
| **I/O 密集型** | 10-20 | 充分利用并发 |
| **LLM API 调用** | 3-5 | 避免 rate limit |
| **混合型** | 5-10 | 平衡资源使用 |

### 8.3 错误处理策略

```typescript
// 1. 超时设置
const result = await task_dispatch_ready({
  profile: 'general-purpose',
  max_parallel: 5,
  wait: true,
  wait_timeout_ms: 300000  // 5 分钟
});

// 2. 失败重试（需要手动实现）
for (const item of result.wait_results) {
  if (item.status === 'failed') {
    // 重新创建任务或记录失败
    console.error(`Task ${item.task_id} failed`);
  }
}

// 3. 清理孤立运行
await task_gc_runs({
  older_than_hours: 24,
  limit: 500
});
```

### 8.4 监控和告警

```typescript
// 定期检查卡住的任务
const tasks = await task_list({ status: 'running', limit: 100 });
for (const task of tasks) {
  const runs = await service.listRuns(sessionId, { taskId: task.id });
  const runningTooLong = runs.some(r => 
    r.status === 'running' && 
    Date.now() - new Date(r.startedAt).getTime() > 600000  // 10 分钟
  );
  
  if (runningTooLong) {
    // 发送告警或自动取消
    await task_run_cancel({ run_id: runs[0].id });
  }
}
```

## 十、总结

### 10.1 架构亮点

本项目通过 **Task V2** 系统实现了一个企业级的多子代理并行执行框架，其核心亮点包括：

1. **分层架构**: Tool → Service → Runner → Repository 清晰分层
2. **DAG 调度**: 支持复杂依赖关系和循环检测
3. **并发控制**: 可配置的 max_parallel 和智能调度
4. **状态持久化**: SQLite + JSONL 双层存储保证可靠性
5. **故障隔离**: 每个子代理独立上下文和错误边界
6. **可观测性**: 完整的事件追踪和状态监控

### 10.2 适用场景

- ✅ **大规模代码分析**: 并行分析多个模块
- ✅ **自动化测试**: 并行执行测试套件
- ✅ **数据处理**: 并行处理多个数据源
- ✅ **文档生成**: 并行生成多个模块的文档
- ✅ **代码审查**: 并行审查多个 PR 或文件

### 10.3 改进建议

2. **资源限制**: 添加 CPU/内存资源限制

4. **Web UI**: 添加可视化监控界面
